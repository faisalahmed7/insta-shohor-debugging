(1).How Does JavaScript Work

JavaScript is high level, dynamic and client side scripting language. Basically it runs on the side of users or clients end line by line by their web browsers. Because modern web browsers support JavaScript and JavaScript engine that runs the codes.At first the codes are taken to the engine as a parser. Then it checks any kinds of error line by line in the codes. Then parser makes a AST(Abstract Syntax Tree) if there are no error in the codes.So when the AST is created by the help of the parser, the JavaScript Engine like (v8, Chakra, JavaScriptCore etc.) converts the code into byte code or machine codes.Here the JS engines act like Just In Time Compiler(JIT). So when the JS code is converted to the byte code or machine code, it is sent to the engine for execution and we get the output.

(2).Difference Bewteen Local Storage and Session Storage

Local storage and Session storage both are stored the data as an object key and value. But the main difference is when the session storage data or object will expire when the page will end or the specific tab of the page will be closed. But on the side of the local storage, the data will not be expired or deleted although the page session will be end or the specific tab will be closed. Every time when the page will be opened , the session storage data storing will start again. But on local storage, the data will not stored again until there is no modification or storing new data except those stored data. The session storage syntax is like window.sessionStorage and the local storage syntax is window.localStorage.
